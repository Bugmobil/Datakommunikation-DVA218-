General Git Commands

  • git init Initializes a new Git repository.
  • git clone "url" Clones a repository into a new directory.
  • git status Shows the working directory status, including modified, staged, and untracked files.
  • git add "file" Adds a file to the staging area.
  • git add Adds all files to the staging area.
  • git commit -m "message" Commits staged changes with a descriptive message.
  • git log Shows the commit history.
  • git diff Shows the difference between the working directory and the staging area.
  • git diff --staged Shows the difference between the staging area and the repository.
  • git rm "file" Removes a file from the working directory and stages the removal.
  • git mv "old file" "new file" Renames a file and stages the change.

Branching and Merging

  • git branch Lists all local branches in the current repositor.y
  • git branch "branch name" Creates a new branch.
  • git checkout "branch name" Switches to the specified branch and updates the working directory.
  • git checkout -b "branch name" Creates a new branch and switches to it.
  • git merge "branch name" Merges the specified branch's history into the current branch.
  • git branch -d "branch name": Deletes the specified branch.

Remote Repositories

  • git remote Lists all remote repositories.
  • git remote add "remote-name" "url" Adds a new remote.
  • git remote rm remote-name Removes a remote.
  • git fetch "remote-name" Downloads all history from the remote tracking branches.
  • git merge "remote-name" "branch-name" Merges the remote branch into the current branch to bring it up to date.
  • git push "remote-name" "branch-name" Uploads all local branch commits to GitHub.
  • git pull "remote-name" "branch-name" Downloads bookmark history and incorporates changes.
  • git remote show "remote-name" Shows information about a remote.

Undoing Changes

  • git checkout -- "file" Discards changes in the working directory.
  • git reset HEAD "file" Unstages the file, but preserve its contents.
  • git revert "commit" Creates a new commit that undoes all of the changes made in the specified commit.
  • git reset "commit" Undoes all of the commits after the specified commit, preserving changes locally.
